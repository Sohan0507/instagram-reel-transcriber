Instagram Reel Transcription Platform

Comprehensive Product Requirements (PRD) & Technical Solution Architecture

1. Product Requirements Document (PRD)

This section outlines the business objectives, core product scope, design tenets, and system specifications set for the AI automation web application.

1.1 Objectives & Scope

The system was designed to allow marketing managers, social media creators, and copywriters to extract exact text transcripts from Instagram Reels instantly. Rather than performing manual transcription, the platform automates the video downloading, audio processing, and transcription generation, feeding the output directly into a document editor for rapid repurposing.

1.2 Core Product Features

  • Link Ingestion & Validation: Simple, clean input box supporting paste functions. Rigorous client and server-side URL pattern checking to prevent malicious input.
  • Automated Processing Pipeline: Staged tracking displays current operation states (Downloading Reel, Extracting Audio, Transcribing Speech) to provide clear visual feedback to the user.
  • Document Rich Text Editor: Integrated WYSIWYG editor containing controls for bolding, italicizing, lists, blockquotes, and headings, giving users full control to format their transcriptions directly in the browser.
  • Document Export Tools: Immediate exporting utilities that convert the edited HTML document into a plain text (.txt) file or structured Microsoft Word document (.docx) directly on the client side.
  • Dynamic Settings: Floating settings modal allowing clients to use their own personal OpenAI API keys, preventing global API key credit exhaustion.

2. Solution Architecture & Technical Design

This section details the architectural pipeline and files created to resolve complex deployment constraints, such as external drive formatting limitations.

2.1 Ingestion & Processing Pipeline

The backend pipeline is housed in `src/app/api/transcribe/route.ts`. When a POST request arrives with a validated Instagram URL, the server executes the following sequence:

Backend Architecture Sequence:
  1. Generates random, collision-safe IDs to define unique temp file directories.
  2. Spawns a child process running `yt-dlp` to download the reel to the system's temporary directory.
  3. Spawns an `ffmpeg` child process to strip video data and output a 16kHz mono, 128kbps MP3 stream (optimized to keep upload payload size tiny).
  4. Uploads the audio stream directly to OpenAI's Whisper API (`whisper-1`) using either the server's private key or a client-provided API key from the request headers.
  5. Performs strict deletion cleanup in a finally {} block to guarantee no raw media logs or files persist on the disk.

2.2 Rich Text & Export Core

  • Editor (src/components/Editor.tsx): Restructures Tiptap StarterKit presets to load styling and button active states. CSS styling classes are appended in `src/app/globals.css` to wrap elements inside the editor (such as lists or blockquotes) with proper margins, spacing, and borders.
  • Client-Side Exporting (src/utils/export.ts): Bypasses server-side document compilers by parsing the raw editor HTML directly inside the client's browser.
    • TXT: Traverses the HTML nodes, formatting paragraph breaks and bullets, and generates a client-side Blob downloaded as a `.txt` file.
    • DOCX: Uses the `docx` package to map headings, bold text, italics, blockquotes, and lists into Microsoft Word XML runs, compiling them into a `.docx` file for download directly in the client's browser.

2.3 Filesystem Layer Patch (exFAT Support)

During local development, running Next.js compiler steps on filesystems that do not record file ownership or support symlinking (such as **exFAT** external drives on Windows) throws an EISDIR: readlink crash because Node's filesystem module receives unexpected return codes.

To solve this, we created **`patch-fs.js`** in the project root. This script monkey-patches Node's filesystem APIs:

const fs = require('fs');
const originalReadlink = fs.readlink;

// Intercepts and maps exFAT readlink directory exceptions into standard EINVAL
fs.readlink = function (path, options, callback) {
  // ... maps EISDIR and UNKNOWN error codes to EINVAL ...
};

This allows the project to build and compile on any filesystem format without modification.

3. Deployment & Operations Guide

Operational instructions to configure, run, and host the platform permanently for free.

3.1 Local Environment Run

To run the application locally on a development machine:

  1. Install dependencies: npm install --legacy-peer-deps
  2. Configure a local file named .env.local containing: OPENAI_API_KEY=sk-...
  3. Run the custom development server script (to bypass exFAT symlink issues): 
    node -r ./patch-fs.js node_modules/next/dist/bin/next dev
  4. Open http://localhost:3000 in your browser.

3.2 Cloud Containerization (Dockerfile)

Because the app requires system binaries (`yt-dlp` and `ffmpeg`), standard serverless platforms like Vercel cannot host it. Instead, we package it inside a Docker container. The included Dockerfile sets up a lightweight Debian image, downloads Python, installs `ffmpeg`, downloads the latest `yt-dlp` binary, and builds the Next.js app.

To deploy to Render:

  1. Create a Web Service pointing to your GitHub repository.
  2. Select **Docker** as the Runtime (Render will automatically read the `Dockerfile`).
  3. Select the **Free Plan**.
  4. Click **Deploy Web Service**.

3.3 24/7 Free Execution (Uptime Robot Trick)

The Render Free Plan automatically puts containers to "sleep" after 15 minutes of inactivity. When a user opens the site during sleep, they experience a 50-90 second "cold start" delay.

To keep the site awake 24/7 for free:

  1. Copy your Render website URL (e.g. https://your-app.onrender.com).
  2. Sign up for a free account at **[UptimeRobot.com](https://uptimerobot.com/)**.
  3. Click **Add New Monitor**.
  4. Set the Monitor Type to HTTPS, enter your URL, and set the interval to 10 minutes.
  5. Save the monitor. UptimeRobot will ping the app every 10 minutes, keeping the container active and ready to load instantly 24/7.
